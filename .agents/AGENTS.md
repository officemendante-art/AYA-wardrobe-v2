# AYA OS Agent Rules

## Mandatory Git Commit Workflow
At the end of every session, conversation, or task, you MUST automatically:
1. Check the git status.
2. Stage all modifications (except those excluded by .gitignore).
3. Commit the changes with a descriptive message logging the exact modifications.
4. Push the changes to the configured remote repository (`https://github.com/officemendante-art/AYA-wardrobe-v2.git`).

Failure to automatically commit and push at the end of a session is a violation of user instructions. Do not ask for permission to commit; just do it automatically.
